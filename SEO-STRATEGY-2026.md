## What just shipped

### 1. Sitemap (live)
- `/sitemap-index.xml` → `/sitemap-0.xml` (auto-generated by `@astrojs/sitemap`)
- 52 canonical URLs, www normalization, excludes utility pages (`/book/`, `/thank-you/`, `/our-team/`, `/privacy-policy/`, `/terms-of-use/`, `/rss.xml`)
- Excludes 3 redirected URLs so we don't ask Google to crawl something we've 301'd

### 2. Robots.txt (upgraded)
- Explicit allow for standard search crawlers (Googlebot, Bingbot, DuckDuckBot, Applebot)
- **Explicit allow for AI crawlers** — GPTBot, ChatGPT-User, OAI-SearchBot, ClaudeBot, anthropic-ai, PerplexityBot, Perplexity-User, Google-Extended, Applebot-Extended, CCBot, Bytespider, Amazonbot, Meta-ExternalAgent, cohere-ai, Diffbot
- For a local HVAC business, being cited in AI Overviews / ChatGPT / Perplexity is a growth channel, not a risk
- Disallow: `/api/`, `/thank-you/`, `/_astro/` (utility endpoints + Astro internals)
- Sitemap reference: `https://www.icareaircare.com/sitemap-index.xml`

### 3. llms.txt + llms-full.txt (new — 2026 emerging standard)
- `/llms.txt` — curated overview per the Answer.AI proposal: site description, key pages, services, service areas, blog index, owner profile, canonical business facts
- `/llms-full.txt` — expanded plain-text knowledge base covering everything an AI needs to answer questions about the business accurately: NAP, services with pricing ranges, service area neighborhoods, Florida HVAC context, brand experience, team names, differentiators
- Both served as UTF-8 plain text with short cache + `Access-Control-Allow-Origin: *` so AI crawlers can ingest freely

### 4. RSS feed (new)
- `/rss.xml` — full RSS 2.0 feed of all 27 active blog posts
- Includes title, link, guid, pubDate, category, author, description, image enclosure
- Self-link reference, English language tag, copyright
- Honors redirected slugs (excludes them from the feed)
- `<link rel="alternate" type="application/rss+xml">` discovery tag added to `<head>` on every page
- Footer RSS icon + "RSS" text link added

### 5. WebSite schema (new)
- Added to the homepage `@graph` alongside LocalBusiness and Person (Tim Hawk)
- `inLanguage: en-US`, publisher linked to LocalBusiness by `@id`
- Gives AI crawlers a clean top-level site description

### 6. _headers (new — Cloudflare Pages security + content-type hygiene)
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: SAMEORIGIN`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: camera=(), microphone=(), geolocation=(), interest-cohort=()`
- `Strict-Transport-Security: max-age=31536000; includeSubDomains; preload`
- Correct Content-Type on llms.txt, robots.txt, sitemap, RSS
- Long cache on hashed `/_astro/*` assets, 30-day cache on images

## Additional 2026 strategies to consider (not yet shipped)

Ranked by impact × effort. Skim and flag what's worth doing.

### High impact / low effort

**1. IndexNow submission (Bing + Yandex instant indexing)**
- Generate a unique key (random 32-char hex), host at `/<key>.txt` in public/, then ping `https://api.indexnow.org/indexnow?url=<url>&key=<key>` on every deploy
- Cloudflare Pages has a native IndexNow integration you can toggle on in the dashboard (Workers & Pages → your project → Settings → Builds → Integrations). Turn it on and Cloudflare auto-submits every updated URL.
- Effort: 5 minutes via Cloudflare dashboard

**2. Google Business Profile → website sync**
- Add a UTM-tagged link from GBP to the site homepage so you can attribute GBP-driven traffic in GA4 (`?utm_source=google&utm_medium=organic&utm_campaign=gbp`)
- Verify GBP name, address, phone, hours match NAP exactly (they do today per this audit)
- Add service attributes in GBP for "Licensed and insured," "On-site services," "Appointment required," etc. These appear in Google local pack.

**3. Schema.org Review snippets from fresh Google reviews**
- Currently emitting hand-curated `Review` objects in `ReviewStrip.astro`. These can go stale.
- Option A: periodically refresh the top 10 reviews from Google Business Profile (manual, every 90 days)
- Option B: set up a Zapier / Make webhook on new GBP reviews → update `src/components/ReviewStrip.astro` → auto-deploy
- Google used to deprecate review stars in organic SERPs for self-published reviews (2019), but `AggregateRating` on LocalBusiness still displays.

**4. FAQ schema on every blog post**
- Most blog posts have implicit Q&A structure but don't emit FAQPage schema per post.
- Easy add: in `src/pages/[slug].astro`, if `post.checklist` exists, also extract H2 + first-paragraph pairs into FAQ items. Would light up FAQ rich snippets for 30 more pages.
- Effort: ~15 min.

**5. `SpeakableSpecification` for voice assistants**
- Tag the first paragraph of service pages + blog posts as speakable:
  ```json
  "speakable": {
    "@type": "SpeakableSpecification",
    "xpath": ["/html/body//h1", "/html/body//p[1]"]
  }
  ```
- Enables Google Assistant / Alexa / Siri to read these aloud for "Hey Google, AC repair near me" queries.

### Medium impact

**6. Google Search Console integration**
- If not connected yet, verify the property at `https://www.icareaircare.com`
- Submit sitemap-index.xml manually first time
- Set up email alerts for coverage issues + Core Web Vitals drops
- The property-level GSC data beats any third-party rank tracker for your own URLs

**7. Microsoft Clarity — behavioral signal mining (already installed via GTM)**
- Once Clarity has 2+ weeks of data, look at rage clicks, dead clicks, scroll depth on service pages. They'll show where the money is actually lost on your 700-reviewers-strong site.

**8. Image sitemap**
- Current sitemap doesn't include images. `@astrojs/sitemap` supports image enrichment via the `serialize` option — attach image URLs per page.
- Images ranking in Google Images is a quiet traffic channel for HVAC (people search "rusted ac coil images" and similar).

**9. Video content + VideoObject schema**
- No video on the site today. Even ONE short "meet Tim" video with VideoObject schema would earn a rich result slot in search.
- Most effective: a 60-second tour of the Wesley Chapel shop + 3-4 testimonials.
- Host on YouTube (use Partytown integration for embed), embed on `/about-us/` with VideoObject schema.

**10. `HowTo` schema wider coverage**
- Currently fires on fix-it posts (`ac-not-cooling`, `warm-air`, `filter-change`, etc.).
- Could extend to: "how to lower cooling bills," "how to book preventive maintenance," "how to find a contractor."
- Every HowTo schema has a shot at the step-by-step rich result (which doesn't compete with regular organic).

### Long-term / higher effort

**11. Generative Engine Optimization (GEO)**
- AI Overviews and ChatGPT cite content differently than Google SERP ranks it. The highest-impact content for GEO:
  - Clear Q&A structure (your service pages have FAQs — good)
  - Specific numbers ("$150–$600" beats "affordable")
  - Named entities with consistent relationships (Tim Hawk → I Care Air Care → Wesley Chapel → Florida license CAC1822037)
  - Original data / unique insights (e.g., "Florida HVAC runs 2,200 hrs/yr vs. Minnesota's 600" — you have this)
- Your llms.txt + llms-full.txt do a lot of this work already.

**12. Backlink acquisition**
- Local citations: Yelp, BBB (if not already listed), Angi, HomeAdvisor, Thumbtack, NextDoor business profile
- Press: Tampa Bay Times local business feature, Patch Wesley Chapel, Tampa Bay Business Journal
- Partnerships: mortgage brokers + realtors refer HVAC pre-purchase inspections (add a rel=dofollow exchange)

**13. Content cluster: "AC repair [neighborhood]" sub-pages**
- You have 9 service-area pages. Consider sub-pages for high-intent neighborhoods within each: "AC repair Seven Oaks," "AC repair Tampa Palms," "AC repair Bexley."
- Risk: cannibalization if done poorly. Only do this for neighborhoods you can actually write 800+ unique words about.

**14. Programmatic SEO — service × area matrix**
- 8 services × 9 areas = 72 possible pages. Most would be thin, but a handful are high-intent ("air duct cleaning Wesley Chapel," "HVAC installation Land O' Lakes") worth building as individual pages with Manual-J numbers, local HOA specifics, and pricing for each area.

**15. Core Web Vitals monitoring**
- Set up Cloudflare Web Analytics (free) or stay with GA4's Web Vitals reporting.
- Target: LCP < 2.5s, INP < 200ms, CLS < 0.1 on 75th percentile mobile.
- Current pages likely already pass (Astro static, WebP images, Partytown for tracking), but worth monitoring as blog + image content grows.

**16. Multi-lingual (Spanish) site**
- Tampa Bay has ~25% Spanish-speaking households. A `/es/` prefix with hreflang tags could open an untapped channel for "reparación de aire acondicionado Wesley Chapel" (~40–70 searches/month per keyword cluster).
- Major effort (full content translation), but meaningful for a bilingual market.
